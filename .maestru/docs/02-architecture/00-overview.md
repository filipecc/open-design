# Architecture Overview

<!-- maestru:summary -->
Open Design is a local-first desktop-class app built as a pnpm monorepo. At runtime it is fundamentally TWO processes: (1) the web app — `apps/web`, a Next.js 16 + React 18 UI fronted by a Node sidecar proxy (`apps/web/sidecar/server.ts`) that serves the UI and forwards `/api/*` to the daemon; and (2) the daemon — `apps/daemon`, a privileged local Node process that owns `/api/*`, agent spawning, skills, design systems, artifacts, SQLite state, and static serving. The browser only ever talks to the web sidecar; the sidecar talks to the daemon over `127.0.0.1`. The agent is NOT shipped — Open Design is BYOK and drives an external coding-agent CLI (Claude Code, Codex, opencode, etc.) or a cloud provider (AMR/Vela) or a raw API key. Generated artifacts (HTML, slides, PDF, images) land in the daemon's data directory (`RUNTIME_DATA_DIR`, default `/app/.od`) and are previewed in a sandboxed iframe. The Electron desktop shell (`apps/desktop`) discovers the web URL via IPC; in dev and in our hosted setup we run the web+daemon pair directly via `tools-dev`.

Open Design is a **pnpm monorepo** that, at runtime, is two cooperating processes plus an external agent.

## The two processes

```
┌─────────┐   HTTP    ┌───────────────────────┐  proxy /api/*  ┌──────────────────────┐
│ Browser │ ────────▶ │  Web sidecar (Node)   │ ─────────────▶ │   Daemon (Node)      │
│ (UI)    │ ◀──────── │  apps/web/sidecar     │  127.0.0.1     │   apps/daemon        │
└─────────┘           │  serves Next.js UI    │ ◀───────────── │  owns /api/*, agents │
                      └───────────────────────┘                │  SQLite, artifacts   │
                                                               └──────────┬───────────┘
                                                                          │ spawns
                                                                          ▼
                                                              ┌──────────────────────┐
                                                              │  Agent CLI (BYO)     │
                                                              │ claude / codex / ...  │
                                                              └──────────────────────┘
```

1. **Web** (`apps/web`) — Next.js 16 App Router + React 18. A Node **sidecar** (`apps/web/sidecar/server.ts`) serves the built UI and **proxies every `/api/*` request to the daemon** over loopback. The browser never contacts the daemon directly. See [10-web/00-overview.md](../10-web/00-overview.md).
2. **Daemon** (`apps/daemon`) — the privileged local process and the `od` binary. It owns the `/api/*` surface, spawns agents, runs skills/plugins, manages design systems and artifacts, holds SQLite state, and serves static assets. See [20-daemon/00-overview.md](../20-daemon/00-overview.md).

The split matters: the daemon is the trust boundary (it can touch the filesystem, spawn processes, hold credentials), and it defends itself with loopback checks and an origin-trust layer. See [03-security-and-origin-model.md](./03-security-and-origin-model.md).

## BYOK — the agent is not included

Open Design ships **no model**. It reads the active design context and drives one of:
- a **local coding-agent CLI** detected on `$PATH` (Claude Code, Codex, opencode, Cursor, Gemini, Qwen… 16+ adapters),
- a **cloud** provider via the AMR/Vela login, or
- a raw **BYOK** API key (Anthropic/OpenAI/Azure/Google/Ollama) proxied by the daemon.

See [30-agents-and-clis/00-overview.md](../30-agents-and-clis/00-overview.md).

## The generation loop

A user fills a question form (surface, audience, tone, brand, scale) → picks a visual direction → the agent streams a plan → the daemon runs **skills** and **design templates** to produce an artifact → the UI previews it in a sandboxed iframe → it can be exported as HTML/PDF/PPTX/ZIP. The content building blocks live in [40-content-system/00-overview.md](../40-content-system/00-overview.md).

## Where things live (top level)

| Path | What |
|------|------|
| `apps/web` | Next.js UI + sidecar proxy |
| `apps/daemon` | Local daemon, `/api/*`, `od` bin |
| `apps/desktop`, `apps/packaged` | Electron shell + packaged runtime |
| `packages/` | Shared libraries (contracts, sidecar, diagnostics…) |
| `skills/`, `design-systems/`, `design-templates/`, `plugins/` | The content system |
| `tools/dev`, `tools/pack`, `tools/serve` | Dev/build/serve control planes |
| `mocks/` | Replay-based mock agent CLIs for tests |

Full breakdown: [04-repo-layout.md](./04-repo-layout.md). The authoritative in-repo source for any subsystem is its own `AGENTS.md`.
